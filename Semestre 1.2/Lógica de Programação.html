Aula 1: Introdução a Algoritmos

Algoritmo é uma sequência de instruções que resolve determinado problema.Uma receita de bolo, por exemplo, pode ser considerada um algoritmo. Veja o detalhamento desse caso a seguir.

Um exemplo de problema: Atravessar uma rua sem ser atropelado.

  1. Olhar para a esquerda
  2. Olhar para a direita
  3. Se estiver vindo algum carro – espere
  4. Repita os passos 1 e 2
  5. Senão estiver vindo carro – atravesse

A algumas formas de fazer algoritmos, essas são as mais famosas: Linguagem Natural, Fluxograma, Diagrama de Chapin, Pseudocódigo (visualG).

Aula 2: Tipos de Dados e Instruções Primitivas

No contexto da computação, dados referem-se a tudo aquilo que é fornecido ao computador de forma bruta. Esses dados são tratados (processados) por um computador, para gerar informações ou resolver problemas.

Dado: é como uma espécie de matéria prima da informação e que, de forma isolada, não traz conhecimento

Informação: pode-se dizer que são dados em um contexto, ou seja, dados que foram tratados e analisados trazendo um significado

Instrução: muito utilizada na área de programação e desenvolvimento de sistemas. Pode-se dizer que constitui um conjunto de palavras-chaves (vocabulário) de uma linguagem de programação para o processamento de dados.

Tipo de Dados:

O tipo de um dado define o conjunto de valores ao qual o dado pertence, bem como o conjunto de todas as operações que podem atuar sobre qualquer valor daquele conjunto de valores. Tipos de dados são dados numéricos inteiros, tipo numérico real, dados literais e dados lógicos.

Aula 3: Variáveis e Constantes

Variáveis:

No contexto da programação, uma variável armazena um valor, que pode variar ao longo da execução do programa. Uma variável possui nome, tipo e conteúdo. O nome deve ser único, a fim de identificar de forma única a variável em um determinado algoritmo.

É importante lembrar que uma variável só pode armazenar um valor de cada vez. No entanto, ela pode assumir vários valores distintos do mesmo tipo durante a execução do algoritmo.

Constantes:

As Constantes são fixas, não mudam de valor. No contexto da programação, o valor de uma constante sempre se manterá inalterado em tempo de execução.

Operadores:

Operadores são utilizados para executar cálculos numéricos e relacionar expressões, são eles: Operadores Aritméticos, Relacionais e Lógicos.

Aula 4: Entrada, Processamento e Saída de Dados

Entrada de Dados:

A entrada de dados apresenta-se na forma de atividades de registro de dados, como gravação e edição. Podem ocorrer através de diversas formas, tais como: Teclado, Câmera, Leitor Ótico, Mouse.

Saída de Dados:

Envolve a transferência de dados produzidos por um processo ou após um processamento do mesmo até o seu destino. Como, por exemplo, o resultado mostrado na tela de um algoritmo ou programa. A saída de dados pode ocorrer através de Monitor, Impressora, Caixa de Som, dentre outros.

Processamento:

De maneira geral, o processamento é a ação de converter dados de forma significativa, mais útil e apropriada, gerando uma informação. No contexto da programação, é a execução dos passos para resolver determinado problema. Constitui a execução de cada linha descrita no algoritmo ou programa, bem como a execução de instruções.

Aula 5: Comandos de Decisão

Em programação, essas decisões podem ser chamadas de Estruturas de Decisão, Estruturas de Controle ou Testes Condicionais. Sendo assim, permitem executar um conjunto diferente de comandos, dependendo do resultado de um teste utilizando operadores relacionais. Este resultado pode ser verdadeiro ou falso.

Comando SE

Faz apenas uma verificação, se o resultado deste teste for verdadeiro, os comandos de dentro do bloco serão executados, caso falso, nenhum comando do bloco é executado.

Comando SE-SENAO

Se o resultado for verdadeiro, os comandos dentro do bloco SE serão executados. Caso o resultado seja falso, somente os comandos do bloco SENAO serão executados.

SE-SENAO (Aninhados)

Consiste basicamente em várias estruturas SE-SENAO, uma dentro da outra. Quando a execução chega ao SENAO, outra expressão é testada.

ESCOLHA-CASO (SWITCH-CASE)

O comando ESCOLHA-CASO é outra opção quando se tem vários IF-ELSE aninhadas. A proposta desta estrutura é permitir ir direto ao bloco de código desejado, dependendo do valor de uma variável de verificação.

Aula 6: Estruturas de Repetição

As Estruturas de Repetição são usadas quando tem a necessidade de repetir partes do código várias vezes. Estes mecanismos de repetição são recursos importantes e muito utilizados em algoritmos e em programas.

PARA (FOR)

Estrutura de repetição PARA consiste em empregar uma variável, geralmente um contador, para controlar as repetições a serem executadas.

ENQUANTO (WHILE)

A estrutura de repetição ENQUANTO permite repetir um trecho de código enquanto uma determinada condição for verdadeira.

REPITA-ATE (DO-WHILE)

O bloco de comandos associado a estrutura de repetição REPITA-ATE é executado obrigatoriamente pelo menos uma vez.Como este laço testa a condição de parada depois de executar sua sequência de comandos, esta sequência poderá ser executada uma ou mais vezes.

Aula 7: Vetor/Array

Estrutura Homogênea Unidimensional é uma estrutura de dados muito simples, também conhecida como vetor ou array. Possui apenas uma dimensão e pode armazenar diversas variáveis do mesmo tipo. Cada item (ou elemento) do vetor pode ser acessado por um índice.

É necessário declarar o vetor em uma seção de variáveis. Variável é o nome dado ao seu vetor seguido de dois pontos. A palavra vetor é reservada e indica que esta variável é um vetor com valor inicial e valor final de um tipo (separados por dois pontos), que pode ser inteiro, real, caractere.

Aula 8: Matriz

Uma matriz é uma variável composta, homogênea e multidimensional, formada por uma sequência de variáveis do mesmo tipo, com o mesmo identificador (mesmo nome) e alocadas sequencialmente na memória.

As matrizes são comumente referenciadas através de suas dimensões, ou seja, são referenciadas pelas quantidades de linhas e colunas.

Sua notação comum é MxN, onde: M é a dimensão horizontal (quantidade de linhas). N é dimensão vertical (quantidade de colunas)

Aula 9: Procedimentos - sem Parâmetros

Modularização

Modularização é a divisão do código em partes ou módulos. Estes módulos são blocos de instruções que realizam tarefas específicas. Uma vez carregado, o módulo pode ser executado quantas vezes for necessário. Além disso, pode ser usado para economizar espaço e tempo de programação, já que pode ser chamado em várias partes de um mesmo programa.

Um procedimento é um bloco de código que executa uma tarefa específica. Quando um procedimento não requer entrada de dados para realizar sua tarefa, ele é considerado um procedimento sem parâmetros. Ele é chamado simplesmente pelo seu nome e executa sua tarefa interna. Exemplo:

        procedimento saudacao():
            escrever("Olá! Como você está?")
        fim_procedimento

Aula 10: Procedimentos - Parâmetros

Às vezes, um procedimento precisa de informações específicas para executar sua tarefa. Essas informações são passadas para o procedimento através de parâmetros. Os parâmetros são variáveis que recebem valores quando o procedimento é chamado e podem ser usados dentro do procedimento. Exemplo:

        procedimento saudacao(nome):
            escrever("Olá, ", nome, "! Como você está?")
        fim_procedimento

Aula 11: Funções sem Parâmetros

Uma função é similar a um procedimento, mas retorna um valor. Assim como um procedimento, quando uma função não requer entrada de dados para produzir seu resultado, ela é considerada uma função sem parâmetros. Você chama a função pelo seu nome e ela retorna um valor. Exemplo:

        função gerarNumeroAleatorio():
            retornar número_aleatório
        fim_função

Aula 12: Função com Parâmetros

Da mesma forma, uma função pode receber parâmetros para produzir seu resultado. Os parâmetros são passados para a função quando ela é chamada e podem ser usados dentro da função para realizar cálculos ou outras operações. Exemplo:

        função calcularSoma(a, b):
            retornar a + b
        fim_função

Aula 13: Visão Geral da Linguagem Java

O Java é uma linguagem de programação amplamente utilizada, conhecida por sua segurança, confiabilidade e versatilidade. Ele é essencial para muitas aplicações, desde sites até supercomputadores científicos e telefones celulares. Para programar em Java, você precisa do JDK (Java Development Kit), que inclui o compilador, o interpretador e o JRE (Java Runtime Environment). A linguagem Java é compilada para um código intermediário e depois interpretada pela JVM (Java Virtual Machine), tornando-a independente do sistema operacional. Escrever e executar programas Java pode ser feito com simples ferramentas como bloco de notas e prompt de comando.

Aula 14: Introdução à Linguagem Java

A linguagem de programação é formal e permite que um programador escreva um conjunto de instruções para criação de programas que manipulam o comportamento físico e lógico de uma máquina. É a comunicação entre o hardware (máquina) e o programador. Seu funcionamento é feito por através de uma série de instruções, palavras-chaves, regras semânticas e sintáticas e símbolos.

Aula 15: Estrutura de Controle no Java

As estruturas de controle em Java são fundamentais para direcionar o fluxo de execução de um programa, permitindo a criação de lógica complexa e a tomada de decisões dinâmicas. Existem três principais categorias de estruturas de controle: condicionais, de repetição e de interrupção.

Condicionais:

Estas estruturas permitem que o programa tome decisões com base em condições específicas. As principais são:

if else:
Executa um bloco de código se uma condição for verdadeira. Se for falsa, executa outro bloco de código
switch:
Permite a seleção entre várias opções, executando o bloco de código correspondente ao caso que corresponda ao valor da expressão.

Repetição (loops):

Estas estruturas permitem a execução repetida de um bloco de código enquanto uma condição for verdadeira.São elas:

for:
Ideal para quando o número de iterações é conhecido. Possui três partes: inicialização, condição e atualização.
while:
Executa um bloco de código enquanto uma condição for verdadeira. A condição é verificada antes de cada iteração.
do-while:
Similar ao while, mas garante que o bloco de código seja executado pelo menos uma vez, pois a condição é verificada após a execução do bloco.

Interrupção:

Estas estruturas são usadas para alterar o fluxo de execução dentro de loops. Como o "break" e o "continue".

Aula 16: Módulos e Matrizes em Java

As matrizes são um conjunto de informações do mesmo tipo. A matriz mais usual é a matriz de uma dimensão que é conhecido como vetor ou array. Você pode desenvolver matrizes de várias dimensões, porém, quanto mais dimensões, mais complicado de entender a organização dos dados.

Para declarar vetor e matriz no java, é assim:

        int vetor[] = {2,4,6,8,10};
        int matriz[][] = new int [2][3];

OBS: o vetor foi criado já com conteúdo e a matriz foi criado só com a sua dimensão que nessa caso é 2 linhas e 3 colunas.
